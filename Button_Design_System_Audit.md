# Button Standardization & Visual Identity Report

## 1. Brand Identity Baseline
- **Primary Color:** `--action-primary` resolves to `--brand-clay`: light `#c2410c`, hover `#9a3412`; dark `#e25822`, hover `#f06c39`. Tailwind utility: `bg-action-primary` / `hover:bg-action-primary-hover`.
- **Secondary Color:** `--action-secondary` resolves to `--brand-olive`: light `#1c3b2b`, hover `#244936`; dark `#345c47`, hover `#427258`. Tailwind utility: `bg-action-secondary` / `hover:bg-action-secondary-hover`.
- **Border Radius Strategy:** Current majority is `rounded-xl` (rounded-xl (289), rounded-lg (152), rounded-2xl (44), rounded-full (18), rounded-md (10), rounded (8)). Standardize action controls on `rounded-xl` for default and large buttons, `rounded-lg` for dense/icon buttons, and avoid one-off `rounded-2xl`, `rounded-full`, or unqualified `rounded` unless the control is intentionally circular.

No `tailwind.config.*` file exists in this frontend. This project uses Tailwind v4 with official brand tokens declared in `app/globals.css` inside `@theme`. Accent and support tokens are `bg-surface`, `bg-surface-muted`, `text-ink`, `text-ink-soft`, `border-line`, `bg-danger`, `bg-success`, `bg-warning`, and `bg-info`.

## 2. The Unified Button Blueprint (Proposed Standard)
> This is the professional standard all buttons must adhere to.

Use a single shared `components/ui/Button.tsx` with a small Tailwind utility map. The package does not currently include `class-variance-authority`, `clsx`, or `tailwind-merge`, so the lowest-friction implementation is a typed variant map plus array join. If CVA is later added, these exact class strings should become the source of truth.

### Proposed Variants (Tailwind Classes)
- **Base Styles (All buttons):** `inline-flex min-h-11 items-center justify-center gap-2 rounded-xl px-4 py-2 text-sm font-medium outline-none transition-[background-color,border-color,color,box-shadow,transform] duration-150 ease-out focus-visible:ring-2 focus-visible:ring-action-primary/30 active:scale-[0.98] disabled:pointer-events-none disabled:opacity-50 disabled:active:scale-100`
- **Primary:** `bg-action-primary text-ink-inverse hover:bg-action-primary-hover`
- **Secondary:** `bg-action-secondary text-ink-inverse hover:bg-action-secondary-hover`
- **Outline:** `border border-line bg-surface text-ink-soft hover:bg-surface-muted hover:text-ink`
- **Ghost:** `bg-transparent text-ink-soft hover:bg-surface-muted hover:text-ink`
- **Danger:** `bg-danger text-ink-inverse hover:bg-danger-strong focus-visible:ring-danger/30`

Recommended size map:
- **Small:** `min-h-9 rounded-lg px-3 py-1.5 text-xs`
- **Default:** `min-h-11 rounded-xl px-4 py-2 text-sm`
- **Large:** `min-h-12 rounded-xl px-6 py-3 text-sm`
- **Icon:** `h-10 w-10 rounded-lg p-0`

State requirements are non-negotiable: every variant must include `hover:`, `active:`, `disabled:`, and `focus-visible:ring-2`. Use `text-ink-inverse` instead of `text-white`, `bg-action-primary` instead of `bg-farm-blue` / `bg-terracotta` / hex values, and `bg-danger` instead of raw red utilities for destructive actions.
